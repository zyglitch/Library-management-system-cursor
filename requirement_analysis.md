# 图书管理系统需求分析

## 一、项目背景和系统目标

随着高校教育信息化发展，传统手工方式管理图书已难以满足师生需求。当前图书馆面临图书借阅流程繁琐、信息查询滞后、欠费管理不实时等问题，学生无法便捷获取借阅状态与欠费信息，管理员也难以高效完成盘点与统计。为实现图书借阅、归还、盘点等业务的信息化管理，建立标准化图书信息数据库，支持多维度检索与统计分析，特开发本系统，以提升图书馆管理效率，优化学生借阅体验，推动校园图书管理数字化转型。

  

实现图书借阅、归还、盘点等业务的信息化管理，建立标准化图书信息数据库，支持多维度检索与统计分析。

## 二、业务需求分析

要能够实现可以：

  

1. 学生查阅图书馆里存在的书籍，以及借阅。
2. 学生可以看自己借用的书籍以及其要交还的日期。
3. 学生可以通过系统来续借（计费（存储过程））。
4. 学生可以看是否有欠费。
5. 学生可以对自己欠费的项目缴费。
6. 学生可以归还书籍。
7. 有一个管理员可以查看现在所有的借书情况（数量，时间，哪些人借了书，有多少欠费的情况）。
8. 有教资工的系统。
9. 计费的系统，实现在借书超过一定日期之后开始以以每天来计费（不同的用户其计费的规则是不一样的）。

  

根据图书管理系统的需求分析，可细化以下数据表，以满足业务功能及数据管理需求：

### 1. 图书表（Books）

|字段名|数据类型|说明|
|---|---|---|
|book_id|主键（字符串）|图书唯一标识|
|title|字符串|书名|
|author|字符串|作者|
|category_id|外键（整数）|关联图书分类表，用于分类检索|
|publisher_id|外键（整数）|关联出版社表，记录出版信息|
|publish_date|日期|出版日期|
|isbn|字符串|ISBN 编号|
|total_count|整数|总库存量|
|available_count|整数|可借阅数量（实时更新）|
|status|枚举值|图书状态（在架、借出、损坏、丢失）|
|shelf_location|字符串|书架位置|
|description|文本|图书简介|

### 2. 学生表（Students）

|字段名|数据类型|说明|
|---|---|---|
|student_id|主键（字符串）|学生唯一标识（如学号）|
|name|字符串|姓名|
|class|字符串|班级|
|department|字符串|学院 / 系别|
|phone|字符串|联系方式|
|email|字符串|电子邮箱|
|borrow_count|整数|当前借阅图书数量（用于限制借阅上限）|
|is_overdue|布尔值|是否有逾期未还图书|
|debt_amount|数值|累计欠费金额（实时同步欠费表）|
|register_time|日期|系统注册时间|

### 3. 借阅记录表（BorrowRecords）

|字段名|数据类型|说明|
|---|---|---|
|record_id|主键（整数）|借阅记录唯一标识|
|user_id|字符串|借阅人唯一标识（学生学号或教职工工号）|
|user_type|枚举值|借阅人类型（"学生"/"教职工"）|
|book_id|外键（字符串）|关联图书表，记录借阅图书|
|borrow_date|日期|借阅日期|
|due_date|日期|应还日期（默认根据规则计算，如学生 30 天、教职工 60 天）|
|return_date|日期|实际归还日期（未归还时为 NULL）|
|status|枚举值|借阅状态（借阅中、已归还、逾期）|
|fine_amount|数值|逾期罚款金额（若有）|
|operator_id|外键（整数）|关联管理员表，记录操作人|

### 4. 欠费表（Fees）

|字段名|数据类型|说明|
|---|---|---|
|fee_id|主键（整数）|欠费记录唯一标识|
|user_id|字符串|欠费人唯一标识（学生学号或教职工工号）|
|user_type|枚举值|欠费人类型（"学生"/"教职工"）|
|record_id|外键（整数）|关联借阅记录表（若欠费由逾期产生）|
|fee_type|枚举值|欠费类型（逾期罚款、损坏赔偿、丢失赔偿）|
|amount|数值|欠费金额|
|due_date|日期|应缴费日期|
|pay_date|日期|实际缴费日期（未缴费时为 NULL）|
|status|枚举值|欠费状态（未缴、已缴、豁免）|

### 5. 管理员表（Administrators）

|字段名|数据类型|说明|
|---|---|---|
|admin_id|主键（整数）|管理员唯一标识|
|username|字符串|登录用户名|
|password|字符串（加密）|登录密码（加密存储）|
|name|字符串|管理员姓名|
|role|枚举值|权限角色（超级管理员、普通管理员）|
|phone|字符串|联系方式|
|email|字符串|电子邮箱|
|register_time|日期|入职 / 注册时间|

### 6. 图书分类表（Categories）

|字段名|数据类型|说明|
|---|---|---|
|category_id|主键（整数）|分类唯一标识|
|category_name|字符串|分类名称（如文学、科技、历史）|
|parent_id|外键（整数）|父分类 ID（支持多级分类，如 “科技” 下分 “计算机”“电子”）|
|description|字符串|分类说明|

### 7. 出版社表（Publishers）

|字段名|数据类型|说明|
|---|---|---|
|publisher_id|主键（整数）|出版社唯一标识|
|publisher_name|字符串|出版社名称|
|address|字符串|地址|
|phone|字符串|联系电话|
|email|字符串|电子邮箱|
|website|字符串|官方网站|

### 8. 教职工表（Faculty）

|字段名|数据类型|说明|
|---|---|---|
|faculty_id|主键（字符串）|教职工唯一标识（如工号）|
|name|字符串|姓名|
|department|字符串|所属部门|
|title|字符串|职称（如教授、讲师）|
|phone|字符串|联系方式|
|email|字符串|电子邮箱|
|borrow_count|整数|当前借阅图书数量（用于限制借阅上限）|
|is_overdue|布尔值|是否有逾期未还图书|
|debt_amount|数值|累计欠费金额（实时同步欠费表）|
|register_time|日期|系统注册时间|
